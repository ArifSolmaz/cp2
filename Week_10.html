Introduction to OOP

Object-Oriented Programming (OOP) is a programming paradigm that organizes code around objects that combine data (attributes) and behavior (methods). Instead of writing separate functions and variables scattered throughout your program, OOP bundles related data and functionality together into cohesive units.

Why Object-Oriented Programming?

Before OOP, programmers used procedural programming where data and functions were separate. This worked for small programs but became difficult to manage as programs grew larger. OOP addresses these challenges by providing better organization, reusability, and modeling of real-world concepts.

Creating Classes and Objects

A class is defined using the class keyword followed by the class name. By convention, class names use PascalCase (each word capitalized). Objects are created by calling the class like a function.

class ClassName:
    """Docstring describing the class."""
    # Class body goes here
    pass

# Create an object (instance)
object_name = ClassName()

The __init__ Constructor

The __init__ method (pronounced "dunder init") is a special method that initializes a new object. It's called automatically when you create an instance. This is where you typically set up initial attribute values.

class ClassName:
    def __init__(self, param1, param2):
        self.attribute1 = param1
        self.attribute2 = param2

# Creating an instance calls __init__ automatically
obj = ClassName(value1, value2)

Instance vs Class Attributes

Python distinguishes between instance attributes (unique to each object) and class attributes (shared by all instances). Understanding this difference is crucial for proper OOP design.

Attribute TypeDefinitionScopeAccess
Instance AttributeDefined in __init__ with selfUnique to each objectself.name or obj.name
Class AttributeDefined in class bodyShared by all objectsClassName.name or self.name

Be careful when assigning to class attributes via an instance (e.g., obj.class_attr = value). This creates a new instance attribute that shadows the class attribute, rather than modifying the shared class attribute. Always use ClassName.attr = value to modify class attributes.

Composition

Composition is a design principle where objects contain other objects as attributes. It represents a "has-a" relationship (a Car HAS an Engine) rather than an "is-a" relationship (which is inheritance, covered in Week 11).
